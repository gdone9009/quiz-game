## 1. 프로젝트 개요
본 프로젝트는 파이썬의 객체지향 프로그래밍(OOP) 원리와 JSON 데이터 처리를 학습하기 위한 CLI 기반 퀴즈 시스템입니다. 코드의 동작뿐만 아니라 설계 의도와 단계별 문제 해결 과정을 기록하는 데 중점을 둡니다.

## 1-1. 퀴즈 주제 선정 이유
* **주제**: 영화 & 시리즈 상식
* **선정 이유**: 선정 이유: 사용자가 1~4 사이의 숫자를 입력하는 직관적인 UX를 제공하며, 정답 시 제공되는 부연 설명을 통해 재미와 정보를 동시에 전달합니다.

* **주요 콘텐츠**: <아바타>, <오징어 게임>, <어벤져스>, <007>, <해리포터> 등 대중적인 영화 및 드라마 상식 5문항.

## 🔍 9. 트러블슈팅 (Troubleshooting Record)

🚨 사례 1: AttributeError 발생 및 해결
현상: main() 함수에서 game.run_quiz() 호출 시 'QuizGame' object has no attribute 'run_quiz' 에러 발생.
원인 분석: 메서드 정의 시 클래스 내부의 들여쓰기 수준이 맞지 않아, 해당 함수가 클래스의 멤버 메서드로 인식되지 않음.
해결 조치: 파이썬의 인덴트(Indentation) 규칙을 재검토하여 run_quiz를 클래스 정의 블록 내부로 이동시킴.
학습 내용: 파이썬에서 들여쓰기는 단순한 가독성 도구가 아니라 코드의 논리적 포함 관계를 결정하는 문법적 요소임을 재확인함.


1) 프로젝트 디렉토리 구조 설계 기준
관심사의 분리(SoC): 프로그램 로직(main.py)과 데이터(state.json)를 엄격히 분리했습니다. 이는 데이터가 수천 개로 늘어나더라도 소스 코드를 수정할 필요가 없는 **'데이터 주도 설계'**를 지향한 결과입니다.

사용자 중심 폴더 구성: 루트 폴더에 실행 파일을 두어 접근성을 높이고, 이미지는 별도 폴더로 관리하여 문서의 가독성과 프로젝트의 정갈함을 유지했습니다.

2) 재현 가능성(Reproducibility) 확보 방식
환경 독립성: 특정 절대 경로를 사용하지 않고 파이썬의 os 라이브러리를 통해 상대 경로 기반으로 파일을 로드하도록 설계했습니다. 덕분에 어떤 컴퓨터에서 git clone을 하더라도 추가 설정 없이 바로 실행이 가능합니다.

데이터 복구 로직: load_data 메서드에서 파일이 없을 경우 기본 딕셔너리 구조를 생성하도록 예외 처리를 해두어, 초기 실행 환경에서의 에러를 원천 차단했습니다.

3) 심층 트러블슈팅: 사용자 입력 예외 처리 (8단계 과정)
가설: 사용자가 숫자가 아닌 '문자'를 입력하거나, 보기 범위 밖의 '큰 숫자'를 입력하면 int() 형변환 에러나 논리적 오류가 발생할 것이다.

확인: 실제 테스트 결과 ValueError로 프로그램이 강제 종료되는 것을 확인했습니다.

조치: while True 무한 루프와 try-except 구문을 결합하여, 유효한 값이 들어올 때까지 사용자에게 다시 입력을 요구하는 **'견고한 입력 검증 시스템'**을 구축했습니다. 이는 프로그램의 **신뢰성(Reliability)**을 보장하는 핵심 장치입니다.
